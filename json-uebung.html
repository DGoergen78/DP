Interaktive JSON-Übung

Relationale Tabellen in dokumentenorientierte JSON-Strukturen umwandeln – angelehnt an IHK-Prüfungsaufgaben für Fachinformatiker (Daten- und Prozessanalyse).

Szenario: Fuhrparkverwaltung der LogiTrans GmbH

Die LogiTrans GmbH verwaltet einen Fuhrpark von Lieferfahrzeugen. Die Daten werden von einem relationalen Datenbanksystem auf ein dokumentenorientiertes System (JSON) umgestellt. Ihre Aufgabe: Wandeln Sie die relationalen Tabellen in JSON-Dokumente um.

Fahrzeuge

idkennzeichentypbaujahr
1SB-LT 101Sprinter2021
2SB-LT 202Transporter2019
............

Wartungen

idfahrzeug_iddatumartkosten
112024-03-15Inspektion450.00
212024-08-20Reifenwechsel320.00
322024-05-10Ölwechsel180.00
............
// JSON-Syntaxhilfe // Werttypen: Text, Zahl, Boolean, null, Objekt, Array // Texte in Anführungszeichen { "attributname": wert, "arrayname": [ { "attributname": wert, ... }, ... ] }

Aufgabe 1: Tabelle Wartungen als JSON

Wandeln Sie die Entitäten der Tabelle Wartungen in JSON-Objekt-Notation um. Erstellen Sie die Einträge mit id 1, 2 und 3.

Jede Wartung ist ein Objekt mit den Attributen id, fahrzeug_id, datum, art und kosten. Achten Sie auf die Datentypen: id und fahrzeug_id sind Zahlen, datum und art sind Texte, kosten ist eine Zahl.

Aufgabe 2: Wartungen als eingebettetes Array

Erstellen Sie ein JSON-Dokument für Fahrzeug id=1 (SB-LT 101, Sprinter, 2021) und betten Sie dessen Wartungen als Array ein.

Das Fahrzeug-Objekt enthält id, kennzeichen, typ, baujahr und ein Array "wartungen". Jedes Wartungs-Objekt im Array benötigt id, datum, art und kosten.

Aufgabe 3: Gesamtdokument

Erstellen Sie ein JSON-Objekt mit einem "fahrzeuge"-Array, das beide Fahrzeuge (id 1 und 2) mit ihren jeweiligen eingebetteten Wartungen enthält.

Erstellen Sie ein äußeres Objekt mit dem Attribut "fahrzeuge", das ein Array mit zwei Fahrzeug-Objekten enthält. Jedes Fahrzeug hat sein eigenes "wartungen"-Array.
